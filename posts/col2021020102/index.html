探索 Hugo - 每日进步一点点

如果每日去写完md 文件,还需要进行hugo 生成并且上传到github 上.虽然只要只要几条命令,但多少还是
** 麻烦** 希望能够用一个脚本去完成. 这样就可以做到优化. 大约搜了一下,修改了一下. 我把这个文件放在站点目录的上一级.

#!/bin/sh
# If a command fails then the deploy stops
set -e
# Build the project.
hugo # if using a theme, replace with `hugo -t <YOURTHEME>`
# Go To Public folder
cd public
# Add changes to git.
git add .
# Commit changes.
msg="rebuilding site $(date)"
if [ -n "$*" ]; then
    msg="$*"
fi
git commit -m "$msg"
# Push source and build repos.
git push origin master
# Pull all .
# git fetch --all && git reset --hard origin/master && git pull
